# Finance Tracker

## 📌 Описание проекта
**Finance Tracker** — это кроссплатформенное приложение для учёта личных финансов, разработанное в рамках курсового проекта.  
Приложение позволяет вести учёт доходов и расходов, управлять категориями, целями накоплений и бюджетами, а также анализировать финансовую статистику за выбранный период.

Проект построен с использованием архитектурного паттерна **MVVM**, что обеспечивает разделение логики, данных и пользовательского интерфейса.
 
---

## 🚀 Основной функционал

### 💰 Операции
- Добавление, редактирование и удаление финансовых операций
- Разделение операций на доходы и расходы
- Фильтрация списка операций
- Валидация вводимых данных

### 🗂 Категории
- Управление категориями доходов и расходов
- Редактирование и удаление через свайп-меню
- Защита от добавления пустых значений

### 🎯 Цели накоплений
- Создание и редактирование финансовых целей
- Отображение прогресса достижения цели
- Уведомление о достижении цели
- Уведомление о просроченной цели

### 📊 Бюджет
- Создание бюджета на период
- Установка лимитов по категориям
- Уведомление о превышении общего бюджета
- Уведомление о превышении лимита по категории

### 📈 Статистика
- Отображение доходов и расходов за выбранный период
- Статистика по категориям
- Прогресс-бары для наглядного анализа
- Автоматическое обновление при смене дат

## 🧱 Архитектура проекта

Проект разделён на несколько логических слоёв:
FinanceTracker.Core
├── Models // Модели данных
├── Services // Бизнес-логика
├── ViewModels // ViewModel (MVVM)
├── Data // Работа с БД (EF Core)
└── Common // Общие утилиты и команды

FinanceTracker.MAUI
├── Views // XAML-страницы
├── Converters // Конвертеры для UI
└── Resources // Стили и ресурсы


- **Core** не зависит от UI и может быть использован с WPF или MAUI
- **MAUI** отвечает только за отображение и взаимодействие с пользователем

## ✔ Валидация данных

Во всех формах реализована валидация:
- невозможность сохранения пустых полей
- проверка корректности числовых значений
- блокировка кнопок при невалидных данных
- визуальные сообщения об ошибках

Валидация выполняется на уровне **ViewModel**.
